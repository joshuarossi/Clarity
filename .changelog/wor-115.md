## WOR-115

Added the **Invite Accept** page (`/invite/:token`) — the landing experience
for invited parties. Logged-out visitors see a centered card explaining Clarity
with a sign-in prompt; logged-in users see the initiator's topic and category
with Accept/Decline buttons. Accepting redeems the token and routes to the
invitee's private intake form; declining closes the case. Already-consumed
tokens show an error with navigation options. The invite token persists through
the authentication flow so the invitation is never lost.
