## WOR-100 — Privacy response filter

Added `convex/lib/privacyFilter.ts`, a pure server-side filter that
detects near-verbatim leaks of one party's private messages in
AI-generated output before it reaches the other party. The filter
tokenizes text on whitespace/punctuation boundaries and flags any run of
8 or more consecutive matching tokens (case-insensitive). Callers
(Synthesis and Coach actions) pass in the candidate text and the other
party's messages; the filter returns `{ passed, matchedSubstring? }` and
leaves retry/fallback logic to the caller.
