# WOR-156

Fixed: Convex deploy was failing because `convex/auth.config.ts` exported full
Auth.js provider objects instead of the `{ domain, applicationID }` shape that
the Convex platform expects. Moved provider configuration (magic-link email and
Google OAuth) into `convex/auth.ts` and simplified the config file to the
deploy-compatible format, unblocking CI e2e tests.
