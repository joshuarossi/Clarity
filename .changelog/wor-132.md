# WOR-132 — Admin template CRUD, versioning, and audit logging

Added server-side admin Convex mutations and queries for managing coaching
templates. Admins can create templates (with an initial v1), publish new
immutable versions with monotonic version numbering, and soft-archive
templates — all behind a `requireAdmin` server-side guard. Every write
operation records an audit log entry (TEMPLATE_CREATED, TEMPLATE_PUBLISHED,
TEMPLATE_ARCHIVED) for full traceability. Existing cases pinned to older
template versions are never affected by new publications or archival.
