# WOR-132 — Admin template CRUD, versioning & audit logging

Added the server-side admin module (`convex/admin.ts`) for managing coaching
templates. Admins can create templates (with an automatic v1), publish new
immutable versions with monotonic numbering, and soft-archive templates —
all behind a `requireAdmin` server-side guard. Every mutation writes an
audit log entry (TEMPLATE_CREATED, TEMPLATE_PUBLISHED, TEMPLATE_ARCHIVED)
for full lifecycle traceability. Existing cases pinned to older template
versions are never affected by new publications or archival.
