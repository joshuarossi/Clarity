## WOR-112

Added the Invites module (`convex/invites.ts`) for secure invite-link
lifecycle management. Case creators now generate a 32-character
crypto-random, URL-safe invite token at case creation time. The new
`invites/redeem` mutation atomically binds the invited party to the case,
creates their `partyStates` row with role `INVITEE`, and marks the token
as consumed — all within a single Convex transaction. Reusing a consumed
token returns a `TOKEN_INVALID` error, and an initiator attempting to
redeem their own invite receives a `CONFLICT` error.
