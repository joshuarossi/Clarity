## WOR-164: Fix "Invalid ID length" error in authenticated endpoints

Authentication helpers (`requireAuth`, `requireAdmin`, `users.me`) now use
`getAuthUserId()` from `@convex-dev/auth/server` to resolve the current
user's document ID, instead of casting the raw `identity.subject` string.
This fixes the "Invalid ID length 65" crash that occurred when users signed
in via magic link or Google OAuth.
