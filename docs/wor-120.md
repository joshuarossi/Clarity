# WOR-120 — Solo Mode: PartyToggle & useSoloActingParty

## Overview

Solo mode allows a single user to play both the Initiator and Invitee roles
within the same browser session. This is primarily useful for developers
testing the full mediation flow and for demo scenarios where a second
participant is not available.

## Components

### `PartyToggle` (`src/components/layout/PartyToggle.tsx`)

A segmented control rendered at the top of the private coaching page on
solo cases (`isSolo === true`). It displays:

- A **"VIEWING AS"** label in uppercase above the buttons (styled with
  `--coach-accent`).
- Two buttons showing the initiator and invitee names. The active button
  receives a surface-lift style (`--bg-surface`, `--shadow-1`).

CSS classes used: `.party-toggle`, `.party-toggle-label`,
`.party-toggle-btn`, `.party-toggle-btn[data-active="true"]` (defined in
`src/styles/components.css`).

### `useSoloActingParty` hook (`src/hooks/useSoloActingParty.ts`)

Returns the current acting role based on the URL query parameter `?as`.

```ts
const { actingRole, isSolo, setActingParty } = useSoloActingParty(caseId);
```

| Field            | Type                                | Description                          |
|------------------|-------------------------------------|--------------------------------------|
| `actingRole`     | `"INITIATOR" \| "INVITEE"`         | Currently selected role              |
| `isSolo`         | `boolean`                           | Whether the case is a solo case      |
| `setActingParty` | `(p: "initiator"\|"invitee") => void` | Updates the URL `?as` param        |

Default is `"INITIATOR"` when the `?as` param is absent.

## URL State

The toggle value is stored in the `?as` query parameter via React Router's
`useSearchParams`. Accepted values are `initiator` and `invitee`. This
ensures the selection survives page refreshes and can be shared via URL.

## Data Isolation

When solo mode is active, all data-fetching components (private coaching,
party state, synthesis) use the acting party's user ID returned by
`useSoloActingParty` instead of the logged-in user's ID. The backend
(`convex/privateCoaching.ts`) resolves messages and AI context per-party,
so each side of the toggle sees an independent coaching conversation.

## Visibility

The `PartyToggle` component is only rendered when the case has
`isSolo === true`. On standard two-party cases it does not appear.
