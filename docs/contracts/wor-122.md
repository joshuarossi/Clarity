# Contract: WOR-122 — Synthesis AI action — generate with privacy filter + state transition

## Why this work exists

When both parties complete private coaching, Clarity needs to bridge their separate experiences into a joint session without exposing either party's raw words. This task implements the synthesis generation action — the critical privacy gate between the private and joint phases. It reads both parties' private content server-side, generates per-party synthesis texts via Claude, validates the output, runs the privacy filter to detect cross-party content leakage, and atomically persists results while transitioning the case forward.

## Files and exports

### `convex/synthesis.ts` (modify)

Currently a stub with an empty `generate` internalAction. This file becomes the full synthesis action module. It follows the same patterns as `convex/privateCoaching.ts` (internalAction + internal queries/mutations) but is **non-streaming** and calls the Anthropic SDK via `client.messages.create()` instead of `client.messages.stream()`.

**`generate`** — the main internalAction. Takes `{ caseId }`. Reads case status (must be BOTH_PRIVATE_COACHING), both partyStates (form fields + confirmation), and all privateMessages for both parties. Assembles the SYNTHESIS prompt via `assemblePrompt()` from `convex/lib/prompts.ts`, including both parties' messages and form fields. Calls Claude Sonnet one-shot. Validates JSON output via `validateSynthesisOutput()`. Runs `filterResponse()` from `convex/lib/privacyFilter.ts` cross-wise (forInitiator checked against invitee's messages, forInvitee against initiator's). On filter match, retries up to 2 times. On final failure, uses `GENERIC_FALLBACK_SYNTHESIS` and inserts an auditLog entry. On success (or fallback), calls `persistSynthesisResults` mutation.

**`persistSynthesisResults`** — internalMutation. Writes `synthesisText` and `synthesisGeneratedAt` to each party's partyState row, and patches the case status to `READY_FOR_JOINT` and `updatedAt` — all in one mutation for atomicity.

**`getCase`** — internalQuery for reading the case doc within the action.

**`getAllPartyStates`** — internalQuery that returns all partyStates for the case (both parties), used to read form fields and validate both parties completed coaching.

**`getAllPrivateMessages`** — internalQuery that returns all privateMessages for the case (both parties, all roles), used to assemble the prompt context and to feed the privacy filter.

The action supports `CLAUDE_MOCK=true` via the existing `isClaudeMockEnabled()` / `getMockClaudeResponse("SYNTHESIS")` from `convex/lib/claudeMock.ts`. In mock mode, the mock SYNTHESIS response is used directly (it's already valid JSON in the expected format).

### `convex/lib/synthesisValidation.ts` (create)

Pure validation module with no Convex runtime dependencies.

**`validateSynthesisOutput(raw: string)`** — Parses a raw string as JSON. Returns `{ ok: true, data: { forInitiator, forInvitee } }` if the result is an object with exactly two string-valued keys `forInitiator` and `forInvitee`. Returns `{ ok: false, error: string }` for malformed JSON, missing keys, extra keys, or non-string values.

**`SynthesisOutput`** — TypeScript type `{ forInitiator: string; forInvitee: string }`.

**`GENERIC_FALLBACK_SYNTHESIS`** — A constant `SynthesisOutput` with generic, safe fallback text for both parties. Used when all retries are exhausted after privacy filter failures.

## Data dependencies

### `internal.synthesis.getCase`
Reads the `cases` document. The action checks `status === "BOTH_PRIVATE_COACHING"` before proceeding. On success, `persistSynthesisResults` patches status to `"READY_FOR_JOINT"` and `updatedAt`.

### `internal.synthesis.getAllPartyStates`
Returns both partyState rows for the case. The action reads:
- `role` (INITIATOR / INVITEE) — to know which synthesis text goes where
- `mainTopic`, `description`, `desiredOutcome` — form fields fed into the prompt context
- `privateCoachingCompletedAt` — validated to be non-null for both parties
- `_id` — used by `persistSynthesisResults` to patch `synthesisText` and `synthesisGeneratedAt`

### `internal.synthesis.getAllPrivateMessages`
Returns all privateMessages for the case. The action uses:
- Messages grouped by `userId` (or `partyRole` for solo cases) to separate initiator's vs invitee's messages
- `role` (USER / AI) and `content` — fed into `assemblePrompt()` as `actingPartyPrivateMessages` and `otherPartyPrivateMessages`
- `content` of USER-role messages per party — fed into `filterResponse()` as `otherPartyMessages`
- `status === "COMPLETE"` — only completed messages are included (not STREAMING or ERROR)

### `assemblePrompt()` (from `convex/lib/prompts.ts`)
Called with `role: "SYNTHESIS"`. The existing assembler builds a system prompt containing `ANTI_QUOTATION_INSTRUCTION` and `SYNTHESIS_OUTPUT_FORMAT_INSTRUCTION`, then structures both parties' messages as context. No template is applied for SYNTHESIS role.

### `filterResponse()` (from `convex/lib/privacyFilter.ts`)
Called twice: once for `forInitiator` against invitee's USER messages, once for `forInvitee` against initiator's USER messages. Returns `{ passed: boolean, matchedSubstring?: string }`.

### `auditLog` table
On final filter failure, an entry is inserted with `action: "SYNTHESIS_PRIVACY_FAILURE"`, `targetType: "case"`, `targetId: caseId`, and metadata containing the matched substrings and retry count. The `actorUserId` is set to the case's `initiatorUserId` (as a system-initiated action, the initiator is the nominal actor since they created the case).

## Invariants

### Cross-party privacy
Both parties' private messages are read server-side within the action. This content is passed to Claude for synthesis but is never returned to any client-facing query. The only client-visible output is the per-party `synthesisText` stored on each party's partyState — and that text has been validated by the privacy filter before persistence.

### Cross-wise filter checking
`forInitiator` (the text the initiator will read) is checked against the **invitee's** private messages. `forInvitee` is checked against the **initiator's** private messages. This ensures neither party's synthesis leaks the other party's raw words. Only USER-role messages are checked (AI responses are not private content from the party).

### Atomic persistence
`persistSynthesisResults` writes to both partyState rows and patches the case status in a single Convex mutation. If any write fails, the entire mutation rolls back. There is no intermediate state where one party has synthesis but the other doesn't, or where synthesis is written but the case status hasn't changed.

### Status precondition
The action validates that `case.status === "BOTH_PRIVATE_COACHING"` and that both partyStates have `privateCoachingCompletedAt` set. If not, the action throws a CONFLICT error and does not call Claude. This prevents double-generation or premature generation.

### One-shot, non-streaming
The action uses `client.messages.create()` (not `.stream()`). There is no STREAMING message row, no periodic flushing, no `stream.on("text")` handler. The full response is received, validated, filtered, then persisted in one go.

### Fallback + audit on final failure
If the privacy filter fails on all 3 attempts (initial + 2 retries), the action writes `GENERIC_FALLBACK_SYNTHESIS` text to both partyStates and inserts an auditLog entry. The case still transitions to `READY_FOR_JOINT` — the fallback is safe generic guidance, not a failure state.

## Edge cases

### Loading state
While the action is in-flight, the case remains in `BOTH_PRIVATE_COACHING` status with both parties' `privateCoachingCompletedAt` set but `synthesisText` / `synthesisGeneratedAt` still null. The frontend can detect this state (both coaching complete, no synthesis yet) and display "Generating your guidance..." without any new fields or queries. When `persistSynthesisResults` runs, the reactive query on `partyStates` fires and the UI updates.

### Mock mode
When `CLAUDE_MOCK=true`, the action uses `getMockClaudeResponse("SYNTHESIS")` which returns pre-formatted valid JSON with `forInitiator` and `forInvitee` keys. Mock mode skips the real API call but still runs validation and the privacy filter against the mock response. Mock failure simulation (`CLAUDE_MOCK_FAIL_COUNT`) is supported for retry testing.

### Solo mode
In solo cases, both partyStates have the same `userId`. The action must group messages by `partyRole` field (not by `userId`) to correctly separate initiator's vs invitee's messages for prompt assembly and filter checking.

### API key missing
If `CLAUDE_MOCK` is not enabled and `ANTHROPIC_API_KEY` is not set, the action logs an error and throws an `AI_ERROR`. No partial writes occur.

### Malformed Claude response
If Claude returns text that doesn't parse as valid JSON or doesn't have the expected shape, `validateSynthesisOutput` returns `{ ok: false }`. The action treats this the same as a filter failure and retries. After exhausting retries, it falls back to `GENERIC_FALLBACK_SYNTHESIS`.

### Case already past BOTH_PRIVATE_COACHING
If `generate` is called but the case has already been transitioned (e.g., due to a duplicate scheduler call), the status precondition check fails with a CONFLICT error. No writes, no Claude call.

## Non-goals

- **No frontend/UI work.** The loading state ("Generating your guidance...") is derived from existing reactive queries on `partyStates` fields. No new components, pages, hooks, or queries are needed for this ticket.
- **No changes to `convex/lib/privacyFilter.ts`.** The filter is consumed as-is. Its `filterResponse()` and `tokenize()` exports are stable.
- **No changes to `convex/lib/prompts.ts`.** The SYNTHESIS role assembler already exists with the correct system prompt, anti-quotation instruction, and output format instruction.
- **No streaming.** One-shot per TechSpec TQ3. No STREAMING message rows, no periodic DB flushes.
- **No template application.** The SYNTHESIS role explicitly does not use templates, per the existing `appendTemplate()` guard in `prompts.ts`.

## Test coverage

### `tests/unit/synthesis.test.ts` — Unit tests (vitest + convex-test)

Covers 7 of 8 ACs:

- **Context assembly AC:** Integration test using `convex-test`. Seeds two users with completed private coaching (populated form fields, multiple private messages each). Runs `generate` action. Asserts that `assemblePrompt` receives both parties' messages and form fields. Verifies the prompt includes messages from both users, not just one.

- **Anti-quotation instruction AC:** Unit test. Asserts that the SYNTHESIS system prompt (as assembled by `assemblePrompt({ role: "SYNTHESIS", ... })`) contains the exact `ANTI_QUOTATION_INSTRUCTION` string.

- **JSON validation AC:** Unit tests of `validateSynthesisOutput()`. Cases: valid `{forInitiator, forInvitee}` passes; malformed JSON rejects; missing keys reject; extra keys reject; non-string values reject; empty strings pass (valid but degenerate).

- **Cross-party filter AC:** Unit test + integration test. Unit: calls `filterResponse()` with synthesis text containing 8+ consecutive tokens from the other party's messages — asserts filter fails. Integration: seeds a case, mocks Claude to return text with known overlap, asserts the action retries.

- **Retry + fallback AC:** Unit test. Mocks Claude to return filter-failing responses 3 times consecutively. Asserts: (1) exactly 3 Claude calls are made (initial + 2 retries), (2) fallback synthesis text is written to both partyStates, (3) auditLog entry is inserted with `action: "SYNTHESIS_PRIVACY_FAILURE"`. Also tests: first filter match triggers retry, second attempt returns clean — asserts clean text is persisted, no audit log entry.

- **Atomic persistence AC:** Integration test using `convex-test`. Seeds a complete case. Runs `generate` (mock mode). Asserts both partyStates have `synthesisText` and `synthesisGeneratedAt` set, case status is `READY_FOR_JOINT`, and `updatedAt` is refreshed.

- **One-shot non-streaming AC:** Unit test. When not in mock mode, asserts the Anthropic SDK is called via `client.messages.create()` (not `.stream()`). In mock mode, asserts `getMockClaudeResponse("SYNTHESIS")` is used and no streaming infrastructure is invoked.
