# Contract: WOR-142 — Fix test assertion for ForbiddenRedirect navigate state

## Why this work exists

WOR-121 implemented the CaseDetail orchestrator page. During review, a HIGH-severity finding identified that ForbiddenRedirect should pass error state to `navigate()` so users see feedback on redirect. The component was correctly updated to pass `{ state: { error: "You do not have access to that case." } }` as the second argument, but the corresponding unit test assertion at line 359 was never updated from the old single-argument form. This causes a test failure that was not resolved before the parent PR merged.

## Files and exports

### `tests/unit/case-detail-page.test.tsx` (modify)

This is the existing unit test file for the CaseDetailPage orchestrator, created by WOR-121. It contains 24 tests covering status routing, phase headers, loading states, and auth guards.

The only change is on line 359: the `toHaveBeenCalledWith` assertion must be updated from:

```typescript
expect(mockNavigate).toHaveBeenCalledWith("/dashboard");
```

to:

```typescript
expect(mockNavigate).toHaveBeenCalledWith("/dashboard", {
  state: { error: "You do not have access to that case." },
});
```

No new exports, no new test cases, no structural changes.
