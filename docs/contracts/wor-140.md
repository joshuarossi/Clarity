# Contract: WOR-140 — Fix tests/unit/seed.test.ts — t.mutation() called with internalMutation

## Why this work exists

The seed test file (`tests/unit/seed.test.ts`) was written before `convex/seed.ts` existed. The test author assumed `seed` would be a public `mutation`, but the implementation correctly made it an `internalMutation` (seed should not be publicly callable). The `convex-test` library's `t.mutation()` method only accepts public mutation references, producing TS2345 type errors at every call site. Additionally, `convex/_generated/api.d.ts` was never regenerated to include the seed module, so `internal.seed.seed` doesn't resolve (TS2339). This ticket fixes both issues so the test file typechecks and runs correctly.

## Files and exports

### `tests/unit/seed.test.ts` (modify)

The only substantive change: replace every `await t.mutation(internal.seed.seed, {})` with `await t.run(async (ctx) => { await ctx.runMutation(internal.seed.seed, {}); })`. This is the convex-test idiom for invoking internal mutations. The file has approximately 18 such call sites across 4 describe blocks (AC1 admin creation, AC2 templates, AC3 idempotency, AC4 production guard).

The import line `import { internal } from "../../convex/_generated/api";` is already present (line 5) and must be retained. The import of `ADMIN_EMAIL` and `DEFAULT_CATEGORIES` from `convex/seed` (line 4) is also retained — these are plain constants, not function references.

Special attention for the production-guard test (line 262): `await expect(t.mutation(internal.seed.seed, {})).rejects.toThrow()` must become `await expect(t.run(async (ctx) => { await ctx.runMutation(internal.seed.seed, {}); })).rejects.toThrow()`. The error thrown inside the mutation handler must propagate through `ctx.runMutation` → `t.run` → the `expect().rejects` chain.

### `convex/_generated/api.d.ts` (modify)

The current generated file does not include the `seed` module in its `fullApi` type map. The seed module must be added so that `internal.seed.seed` resolves to a valid `FunctionReference<"mutation", "internal">`. This requires either:

- Running `npx convex dev` to regenerate (preferred if the dev environment is available), or
- Manually adding `import type * as seed from "../seed.js";` and including `seed: typeof seed` in the `ApiFromModules` generic.

The manual approach is acceptable because this is a generated file that will be overwritten on the next `npx convex dev` run anyway, and the dev agent may not have a running Convex backend.

## Data dependencies

### `internal.seed.seed`

An `internalMutation` with `args: {}` that returns `Promise<void>`. It creates one admin user (`users` table) and three templates with versions (`templates`, `templateVersions` tables) idempotently. Throws if `process.env.IS_PRODUCTION === "true"`. The test file exercises this mutation through `t.run(ctx => ctx.runMutation(...))` and then queries the database directly via `t.run(ctx => ctx.db.query(...))` to verify side effects.

## Invariants

**convex/seed.ts is untouched.** AC4 is explicit: the production code is correct. This ticket changes only the test file and the generated type declarations.

**Mechanical replacement only.** Every `t.mutation(internal.seed.seed, {})` becomes `t.run(async (ctx) => { await ctx.runMutation(internal.seed.seed, {}); })`. No other changes to test logic, assertions, describe/it structure, or imports (beyond what's needed for types). The replacement is 1:1 — same number of seed invocations before and after.

**Error propagation preserved.** The AC4 production-guard tests rely on the mutation throwing. When called via `ctx.runMutation` inside `t.run`, the thrown error must propagate to the outer promise so `.rejects.toThrow()` still works. This is standard behavior — `ctx.runMutation` re-throws mutation errors.

**`internal.seed.seed` must resolve.** After `api.d.ts` is updated, `internal.seed.seed` must have type `FunctionReference<"mutation", "internal">` so that `ctx.runMutation(internal.seed.seed, {})` typechecks.

## Edge cases

**No new edge cases.** This is a mechanical type-fix. The existing tests already cover loading, empty state, idempotency, and the production guard. The only behavioral concern is that `t.run(async (ctx) => { await ctx.runMutation(...) })` propagates errors identically to `t.mutation(...)` — which it does, per convex-test semantics.

## Non-goals

**Do not modify `convex/seed.ts`.** The production code is correct. The `internalMutation` declaration is intentional.

**Do not add new tests.** This ticket fixes existing tests only. No new describe blocks, no new it blocks.

**Do not refactor for DRY.** It would be tempting to extract `const runSeed = (t) => t.run(async (ctx) => { await ctx.runMutation(internal.seed.seed, {}); })` to reduce repetition. Don't — keep the diff mechanical and minimal. If a future ticket wants to DRY up the test, that's separate scope.

**Do not change test structure.** The describe/it hierarchy, assertion logic, and afterEach hooks remain identical.
