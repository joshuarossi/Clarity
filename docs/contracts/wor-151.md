# Contract: WOR-151 — ESLint fails: 4 unused-identifier errors in test files

## Why this work exists

`bun run lint` currently exits non-zero due to 4 `@typescript-eslint/no-unused-vars` errors in test files introduced by recent sibling tickets (WOR-148 for the joint-chat tests, WOR-149 for the message-bubble test). CI lint gates are broken until these are fixed. The fix is trivial dead-code removal — no behavioral change.

## Files and exports

### `tests/unit/joint-chat.test.ts` (modify)

Two test cases destructure variables from `seedJointActiveEnv()` that they never reference in assertions:

- **Line 971** (`AC1` test for WOR-148): `const { t, userAId, userBId, caseId }` — only `t`, `userBId`, and `caseId` are used. Remove the `userAId` binding. Since the destructuring is object-style (not positional), the binding can simply be omitted: `const { t, userBId, caseId }`.

- **Line 995** (`AC2` test for WOR-148): `const { t, userAId, userBId, caseId }` — only `t` and `caseId` are used. Remove both `userAId` and `userBId`: `const { t, caseId }`.

No exports — this is a test file.

### `tests/unit/message-bubble.test.tsx` (modify)

- **Line 8**: `import { MarkdownContent } from "../../src/components/chat/MarkdownContent"` is entirely unused. Remove the entire import statement.

No exports — this is a test file.

## Data dependencies

None. This ticket modifies only test files and touches no queries, mutations, or runtime code.

## Invariants

**No assertion weakening.** The removed bindings are never referenced in `expect()` calls or passed to functions under test. Specifically:

- Line 971's test asserts on `userBId` and `caseId` — `userAId` is unused.
- Line 995's test asserts via `toHaveProperty` on the notification object — neither `userAId` nor `userBId` is compared directly.
- `MarkdownContent` is imported but never rendered or asserted on in message-bubble.test.tsx.

**No eslint suppression.** The fix removes dead code; it does not add `// eslint-disable` comments or weaken the lint configuration.

## Edge cases

None. This is a mechanical deletion of unused bindings. There are no runtime edge cases.

## Non-goals

- Do not fix lint warnings (only errors are in scope).
- Do not touch any file other than the two named test files.
- Do not change eslint configuration or add disable-comments.
- Do not refactor the tests or improve their coverage.

## Test coverage

All 5 ACs are verified by running `bun run lint` (AC1) and `bun run test` (AC5). ACs 2–4 are structural — they are verified by the absence of lint errors on the specific lines. No new test files are needed; the existing test files *are* the files being fixed, and their continued passing (`bun run test`) is the verification of AC5.
