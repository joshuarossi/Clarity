# Contract: WOR-135 — Admin audit log page

## Why this work exists

Admins need a single surface to trace every administrative change back to the actor and timestamp. The audit log page at `/admin/audit` fulfills the day-one audit logging requirement from the TechSpec (§5.1, §7.2). Existing admin mutations (template create/publish/archive from WOR-132) already write to the `auditLog` table — this ticket adds the read surface.

## Files and exports

### `convex/admin.ts` (modify)

Add a new `listAuditLog` paginated query. Uses `requireAdmin` for server-side gating. Supports optional filters: `actor` (actorUserId string), `action` (action type string), `dateFrom`/`dateTo` (numeric timestamps filtering on `_creationTime`). Returns paginated results enriched with actor display name/email resolved from the `users` table.

Uses Convex's built-in `paginationOptsValidator` and `.paginate()` cursor-based pagination. Default page size: 25.

### `convex/schema.ts` (modify)

Add indexes to the `auditLog` table to support efficient filtering:
- `by_action` index on `["action"]` — for action type filtering
- `by_createdAt` index on `["createdAt"]` — for date range filtering

Note: `by_actor` already exists.

### `src/routes/AdminAuditLogPage.tsx` (create)

Connected component. Calls `usePaginatedQuery(api.admin.listAuditLog, filters, { initialNumItems: 25 })`. Contains:
- Filter bar: actor text input, action type select dropdown, date-from and date-to inputs
- Table with columns: Actor (display name/email), Action, Target (targetType:targetId), Timestamp (formatted from `_creationTime`)
- Pagination controls (Load More or page-based navigation using Convex cursor pagination)
- Right-side Sheet (shadcn/ui) drawer opened on row click, showing metadata as pretty-printed JSON in a `<pre>` block with `font-family: 'JetBrains Mono'` per DesignDoc §2.2
- Solo mode entries visually flagged (e.g., a badge or distinct styling if the target case is solo)

This is a **connected** component (not presentational) because it owns its data fetching, filter state, and pagination state. The test author should mock at the Convex query level, not pass props.

### `src/App.tsx` (modify)

Replace the inline `AdminAuditPage` stub function with an import of `AdminAuditLogPage` from `src/routes/AdminAuditLogPage.tsx`. The route definition at `/admin/audit` already exists wrapped in `<AdminRoute>`.

## Data dependencies

### `api.admin.listAuditLog`

Returns a Convex `PaginationResult` shape:
```ts
{
  page: Array<{
    _id: Id<"auditLog">;
    actorUserId: Id<"users">;
    actorDisplayName: string;  // enriched: user.displayName ?? user.email
    action: string;            // e.g. "TEMPLATE_CREATED", "TEMPLATE_PUBLISHED", "TEMPLATE_ARCHIVED"
    targetType: string;        // e.g. "template", "templateVersion"
    targetId: string;
    metadata: any;             // arbitrary JSON stored by the writing mutation
    createdAt: number;         // epoch ms
    _creationTime: number;     // Convex system field
  }>;
  isDone: boolean;
  continueCursor: string;
}
```

Fields consumed by the UI:
- `actorDisplayName` → Actor column
- `action` → Action column
- `targetType` + `targetId` → Target column (rendered as `targetType:targetId`)
- `createdAt` → Timestamp column (formatted as locale date-time string)
- `metadata` → Drawer JSON content (pretty-printed)

### `api.users.me`

Used by the `AdminRoute` wrapper (already exists). Not called directly by this page.

## Invariants

**Server-side admin gate:** Every call to `listAuditLog` invokes `requireAdmin(ctx)` which throws a `ConvexError` with code `FORBIDDEN` if the authenticated user's role is not `ADMIN`. This is the authoritative enforcement; the client-side `AdminRoute` redirect is defense-in-depth.

**Read-only:** The page renders no forms (except filter inputs), no edit buttons, no delete buttons, and calls no mutations. The only Convex function invoked is the `listAuditLog` query.

**Solo mode flagging:** Audit entries whose target is a solo-mode case should be visibly flagged per the security checklist. The query enriches entries with an `isSoloTarget` boolean when `targetType` is "template" or similar (or the UI resolves this — implementation may look up the case). At minimum, the flag is visible in the metadata JSON; ideally a visual badge on the row.

## Edge cases

**Loading state:** While `listAuditLog` is loading (returns `undefined`), display a skeleton table with shimmer rows (consistent with `AdminTemplatesPage` loading pattern).

**Empty state:** If the query returns zero results (either no audit entries exist, or filters match nothing), display a centered message: "No audit log entries found." with appropriate `data-testid="empty-state"`.

**Error state:** Convex query errors surface via the standard React error boundary. No custom error UI needed for this page.

**Filter combinations:** All filters are AND-combined. If a user selects actor + action + date range, only entries matching all three appear. Clearing a filter removes that constraint. Date range is inclusive on both ends.

**Pagination boundary:** When `isDone` is true, the "Load More" / "Next" button is hidden or disabled. Page 1 always loads on mount; subsequent pages load on user interaction.

## Non-goals

- **Editing or deleting audit entries** — the audit log is append-only by design; no mutation surface exists or should be created.
- **Creating audit entries** — handled by existing admin mutations in `convex/admin.ts` (WOR-132). This ticket only reads.
- **Full-text search of metadata** — filtering is limited to the structured top-level fields (actor, action, date). Searching inside the JSON metadata blob is out of scope.
- **CSV/PDF export** — deferred; not in the acceptance criteria.
- **Real-time streaming of new entries** — Convex reactive queries handle this automatically; no special SSE/WebSocket work needed.
