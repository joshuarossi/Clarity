# Contract: WOR-103 — Theme and style setup

## Why this work exists

Clarity's entire UI depends on a consistent design-token foundation. Without this task, every downstream component (T10–T43) would invent its own colors, spacing, and typography decisions, leading to visual drift and expensive rework. This task translates the StyleGuide's decisions into runtime CSS custom properties, reusable class recipes, a TypeScript token mirror, and a shadcn/ui Button override — so that all subsequent UI work references tokens, never hardcoded values.

## Files and exports

### `src/styles/globals.css` (create)

The single source of truth for all design tokens at runtime. Declares CSS custom properties for every color, spacing, radius, shadow, motion, typography, and font-family token from the StyleGuide. Light theme values live under `:root`; dark theme values under `[data-theme="dark"]`. Also sets font smoothing on `html`, declares `@font-face` or `@import` for Google Fonts (Inter 400/500/600, JetBrains Mono 400/500), defines a `:focus-visible` ring utility using `--accent`, and wraps all animation rules in `@media (prefers-reduced-motion: no-preference)`. Imports `./components.css` so consumers only need one import.

### `src/styles/components.css` (create)

Reusable class recipes for patterns that appear across multiple components. Contains all 7 `.cc-bubble*` variants with the correct background/border token references per StyleGuide §6.4, the `@keyframes cc-blink` animation for the streaming cursor, and the `.cc-streaming-cursor` utility class (2px × 1em bar, `animation: cc-blink 1s steps(2, start) infinite`). Every bubble shares: `padding: 12px 16px`, `border-radius: var(--radius-lg)`, `max-width: min(640px, 80%)`, font at chat size (16px/1.55 line-height), and a 150ms fade + 8px y-translate enter animation (applied once, not on scroll).

### `src/styles/theme.ts` (create)

A TypeScript mirror of the token values for use in JS-driven contexts (charts, SVG rendering, email templates). Exports a default object with `light` and `dark` keys, each containing all color token values as strings. Values must exactly match the CSS custom property values in globals.css. This is a static data file — no runtime DOM reading.

### `src/components/ui/button.tsx` (create)

A shadcn/ui-style Button component overridden with Clarity's five variants: `primary` (sage fill, white text), `secondary` (border + subtle background), `ghost` (transparent, hover background), `danger` (terracotta fill), and `link` (inline text styling). Three size variants: `sm` (32px height), `md` (40px height, default), `lg` (48px height), plus `icon` (36×36). Uses `class-variance-authority` (cva) for variant management. Disabled state uses 50% opacity + `cursor: not-allowed` + `pointer-events: none` — never a different fill color. This is a **presentational** component: it takes props and renders markup. It has no internal hooks or data fetching.

### `index.html` (create)

The app's HTML entry point. Contains: (1) an inline `<script>` in `<head>` that reads `localStorage.getItem("theme")`, falls back to `window.matchMedia("(prefers-color-scheme: dark)").matches`, and sets `document.documentElement.dataset.theme` to `"light"` or `"dark"` before any stylesheet loads — this prevents flash of wrong theme; (2) `<link>` tags for Google Fonts (Inter 400,500,600 and JetBrains Mono 400,500) with `rel="preconnect"` for `fonts.googleapis.com` and `fonts.gstatic.com`; (3) the standard Vite `<div id="root">` mount point and `<script type="module" src="/src/main.tsx">` entry.

## Data dependencies

None. This task has no Convex queries, mutations, or actions. It is purely a frontend visual foundation.

## Invariants

**Token parity across themes.** Every CSS custom property declared under `:root` must also be declared under `[data-theme="dark"]`. A token that exists only in one theme would cause undefined `var()` references when the other theme is active.

**Bubble class completeness.** All 7 `.cc-bubble*` classes must exist with their specified background/border combinations. Downstream component tasks (T10 MessageBubble) will reference these classes by name — a missing class means a broken visual.

**No-flash dark mode.** The inline script in `index.html` must run synchronously in `<head>`, before any `<link rel="stylesheet">` or `<script type="module">` tag. If it runs after paint, users see a flash of the wrong theme.

**Reduced motion respect.** All animation and transition declarations must be wrapped in `@media (prefers-reduced-motion: no-preference)` so that users who prefer reduced motion see no animations. This applies to the streaming cursor blink, bubble enter animations, and any transition durations.

**Font smoothing on html.** The three rules (`-webkit-font-smoothing: antialiased`, `-moz-osx-font-smoothing: grayscale`, `text-rendering: optimizeLegibility`) are set on the `html` element in globals.css, not on `body` or `*`, ensuring they cascade to all elements.

**Button variant exhaustiveness.** The Button component must support exactly the 5 named variants (primary, secondary, ghost, danger, link) and 4 size options (sm, md, lg, icon). No other variants should be added — the StyleGuide §6.1 defines the complete set.

**theme.ts ↔ globals.css sync.** The hex values in theme.ts must exactly match the corresponding CSS custom property values. Any drift means JS-rendered visuals (future charts, emails) won't match the CSS-rendered UI.

## Edge cases

**Loading state.** Not applicable — these are static CSS and a static TypeScript file. No async loading beyond Google Fonts, which use `display=swap` to avoid FOIT (flash of invisible text).

**Empty state.** Not applicable — tokens are always present.

**Error state.** If Google Fonts fail to load, Inter and JetBrains Mono fall back to the system font stacks declared in `--font-sans` and `--font-mono` (`system-ui, -apple-system, "Segoe UI", Roboto, sans-serif` and `ui-monospace, SFMono-Regular, Menlo, Consolas, monospace` respectively).

**No localStorage available.** The inline dark-mode script must handle the case where `localStorage` throws (e.g., private browsing in some older browsers). Wrap the `localStorage.getItem` call in a try/catch and fall back to `prefers-color-scheme`.

**No `matchMedia` support.** If `window.matchMedia` is unavailable (extremely unlikely given NFR-BROWSER targets), default to light theme.

## Non-goals

**No Tailwind configuration.** Tailwind CSS is not yet a project dependency. This task establishes CSS custom properties directly. When/if Tailwind is added, its config can reference these same custom properties, but that wiring is out of scope here.

**No theme toggle UI.** This task creates the detection/persistence mechanism (inline script + localStorage). A visible toggle button/switch belongs in a layout or nav component task.

**No components beyond Button.** Other shadcn/ui overrides (Dialog, Input, Textarea, etc.) are out of scope. Downstream tasks will create them as needed.

**No page layouts or routing.** This is purely the visual token foundation. The app shell (main.tsx, App.tsx, routing) belongs to a separate task.
