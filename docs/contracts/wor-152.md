# Contract: WOR-152 — Fix 2 TS2739 errors in case-private-page.test.tsx

## Why this work exists

The `main` branch build is red because `tests/unit/case-private-page.test.tsx` has two object literals (lines 544 and 579) that assign to the `self` property of `PartyStates` but omit the required `role` and `formCompletedAt` fields. TypeScript reports TS2739 for each. The fix is to supply the missing properties consistent with the test's intent.

## Files and exports

### `tests/unit/case-private-page.test.tsx` (modify)

This is an existing unit test file. No exports change. Two `beforeEach` blocks assign incomplete object literals to `partyStatesFixture.self`. Each must be expanded to include:

- `role: "INITIATOR"` (matching `DEFAULT_PARTY_STATES.self.role`)
- `formCompletedAt: NOW - 100_000` (matching `DEFAULT_PARTY_STATES.self.formCompletedAt`, or `null` — either is acceptable; use `NOW - 100_000` to match the default and signal the form was already completed before private coaching finished)

**Line 544** (AC 8 `beforeEach`):

```typescript
// Before (broken):
self: { privateCoachingCompletedAt: NOW },
// After (fixed):
self: { role: "INITIATOR", formCompletedAt: NOW - 100_000, privateCoachingCompletedAt: NOW },
```

**Line 579** (inside the "both complete" test):

```typescript
// Before (broken):
self: { privateCoachingCompletedAt: NOW },
// After (fixed):
self: { role: "INITIATOR", formCompletedAt: NOW - 100_000, privateCoachingCompletedAt: NOW },
```

## Data dependencies

None. This is a test-only fix with no runtime code changes.

## Invariants

- **Build passes**: After the fix, `npx tsc --noEmit` must exit 0.
- **No behavior change**: The tests must continue to pass with the same assertions. Adding the missing fields does not alter the mock's behavior because the tests under AC 8 only care about `privateCoachingCompletedAt` being non-null to trigger the read-only state.

## Edge cases

None. This is a mechanical type-correctness fix.
