# Transcript Compression

> Module: `convex/lib/compression.ts` · Ticket: WOR-101

## Purpose

Clarity's AI actions (Coaching, Synthesis, Joint Facilitation) each
operate within a fixed context-window token budget. As conversations
grow, the raw transcript can exceed that budget. The **transcript
compression module** (TechSpec §6.4) detects when a message array is
over budget and compresses it down by replacing the oldest messages with
a concise summary, preserving critical context without losing the thread
of the conversation.

## API

### `compressTranscript(messages, budgetTokens, options?)`

| Parameter        | Type                    | Description                                                                   |
| ---------------- | ----------------------- | ----------------------------------------------------------------------------- |
| `messages`       | `CompressibleMessage[]` | The conversation messages to compress.                                        |
| `budgetTokens`   | `number`                | Maximum token budget (e.g. 60 000 for generation, 10 000 for classification). |
| `options.client` | `AnthropicClient`       | Anthropic SDK client instance — required when compression fires.              |

Returns `Promise<CompressibleMessage[]>` — a message array that fits
within `budgetTokens`.

**Behaviour:**

- If total estimated tokens ≤ `budgetTokens`, the input array is
  returned unchanged (no API call).
- If over budget, the oldest 50 % of messages are replaced with a single
  system message prefixed `SUMMARY:`, generated by Claude Haiku.

### `estimateTokens(text): number`

Approximates token count using a word-count heuristic (`words / 0.75`).
Slightly over-estimates, which is intentional — it is safer to compress
a little early than to exceed the budget.

### `COMPRESSION_PROMPT`

The exact prompt sent to Haiku:

> Summarize this conversation segment in 500 tokens or fewer, preserving
> facts, decisions, emotional tone, and unresolved threads.

This wording is mandated by TechSpec §6.4 and must not be paraphrased.

## Caching

Summaries are cached by a SHA-256 hash of the concatenated message
content being compressed. The cache is scoped per Anthropic client
instance using a `WeakMap`, so:

- Each client gets its own isolated cache.
- When a client is garbage-collected, its cache is automatically freed.
- Repeated compression of the same message segment returns the cached
  result without an additional Haiku call.

## Summary message format

The synthetic replacement message uses:

```ts
{ role: "system", content: "SUMMARY: <haiku-generated text>" }
```

Downstream consumers can identify compressed segments by checking for
the `SUMMARY:` prefix on system messages.

## Caller contract

The module is a **pure helper** with no Convex runtime dependencies.
Callers must:

1. Supply a valid `AnthropicClient` when messages exceed the budget.
2. Choose the appropriate `budgetTokens` value for their action type
   (60 000 for generation actions, 10 000 for classification actions).
